# Onboarding Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close three onboarding gaps found in review — a stale "Getting started" Help topic, missing wizard reassurance framing, and Android-specific wording shown on web/iOS — with content/copy edits only.

**Architecture:** All changes are in `mobile/`. Help copy lives in `src/constants/helpContent.ts` (exported, unit-tested via `searchHelpTopics`). Wizard step subtitles are literals passed to `WizardScaffold`. Two settings/about screens carry a BYOK blurb. No new components, no logic, no data-model change.

**Tech Stack:** React Native + Expo, TypeScript, Jest (`npm test` in `mobile/`).

## Global Constraints

- Scope is `mobile/` only. Content/copy edits only — no new components, no logic, no dependency changes.
- Wording must be **provider-neutral and platform-neutral**: name Claude (Anthropic) as the *recommendation*, never as the only option; never say "Android" / "Android Keystore" in user-facing copy. Use the phrase **"your device's secure storage"** for the at-rest key location.
- **Leave code comments unchanged** that mention "Android Keystore" / "device keystore" as native-implementation notes (`src/secure/keyStore.ts`, `src/device/clearDeviceData.ts`, `src/device/deviceIdentity.ts`) — accurate and not user-facing.
- All commands run from `mobile/`.

- [ ] **Step 1: Write the failing tests**

Append to `mobile/__tests__/constants/helpContent.test.ts` **below the existing `describe` block** — do NOT re-add the import line; the file already imports `HELP_TOPICS` and `searchHelpTopics` at the top (line 1). Reuse them:

```ts
describe("getting-started topic (onboarding polish)", () => {
  const topic = HELP_TOPICS.find((t) => t.id === "getting-started");

  it("exists", () => {
    expect(topic).toBeDefined();
  });

  it("mentions the two bundled books readable without an account/key", () => {
    const text = JSON.stringify(topic);
    expect(text).toMatch(/two finished books/i);
    expect(text).toMatch(/Library/);
    expect(text).toMatch(/no account or key needed/i);
  });

  it("is provider-neutral — no longer hard-codes 'Add your Anthropic API key'", () => {
    expect(JSON.stringify(topic)).not.toContain("Add your Anthropic API key");
  });

  it("relaunches the signup and key wizard steps via action blocks", () => {
    const steps = (topic?.blocks ?? [])
      .filter((b): b is { kind: "action"; label: string; step: string } => b.kind === "action")
      .map((b) => b.step);
    expect(steps).toEqual(expect.arrayContaining(["signup", "key"]));
  });

  it("is discoverable by a 'books' search", () => {
    expect(searchHelpTopics("books").map((t) => t.id)).toContain("getting-started");
  });
});

describe("no Android-specific wording in Help copy", () => {
  it("no topic body says 'Android Keystore' or 'device keystore'", () => {
    const all = JSON.stringify(HELP_TOPICS);
    expect(all).not.toContain("Android Keystore");
    expect(all).not.toContain("device keystore");
  });
});
```
