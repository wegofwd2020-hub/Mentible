## The Opinion IS the Product

- They explicitly refuse to be a chatbot. The refusal is the value prop.
- Free-form prompting is banned. You get six scope fields and that's it.
- Even Power Mode keeps key:value structure. There is no path back to free-form chat.
- "The LLM is the commodity. The scoping layer is the product IP." That sentence is the moat thesis.
- The default canvas is one textarea and a Generate button. The opinion is so strong it hides itself behind a collapsible side panel.

## BYOK is a Filter, Not Just an Economics Move

- Most BYOK products dodge the Anthropic bill. This one weaponizes BYOK as a filter.
- "Casual / I'll-try-anything-that's-free users" are excluded **by design**, in writing, in `SCOPE.md`.
- Zero token cost also means zero compliance posture as a token reseller. Two wins from one move.
- Adults willing to paste an `sk-ant-` key have already opted into being serious learners.
- The reviewer who sparked the project flagged institutional GTM as expensive. This is the GTM that costs nothing.

## Vendoring is the Honest Move

- They picked vendoring over submodules and over a private package. The reasoning is the wisdom.
- Submodules give you clone-recursive footguns and opaque pointer-only PR diffs. Code review goes blind.
- A shared package would force release-cadence coupling on two products that need to evolve apart.
- "Pretending it's a library is over-engineering" with one consumer. That's anti-YAGNI in plain English.
- `VENDORED.md` tracks each file's source SHA, updates show up as normal PR diffs, and `anthropic.py` is flagged "modified locally" so OnDemand changes trigger a manual three-way merge.

## They Named the Threat "Educational Laundering"

- Most safety docs only talk about jailbreaks. This one names a different vector and labels it.
- The example they wrote is chilling and specific: *"Quiz: home chemistry experiments — for a 9-year-old."*
- It's V4 of five named abuse vectors. Cost-DoS attacks are explicitly out of scope because BYOK shifts the bill onto the attacker.
- Anthropic handles CSAM categorically. The doc says *"that's their layer, not ours."* Honest division of work.
- The five MVP safety defenses (M1–M5) hard-block the override UI from public alpha. Not aspirational, gated.

## What MVP is Allowed to Be Fragile About

- The MVP doc has a section called "What's intentionally fragile in MVP." Most teams hide this list. They led with it.
- No retry on Anthropic timeout. No queue depth limit. No rate limits. No analytics beyond basic structlog.
- "We trust low traffic during the demo phase." That sentence almost never appears in design docs.
- Each install is its own world. No accounts. No cloud sync. They picked the smallest possible end-to-end loop.
- MVP success criteria are exactly six items. When all six pass, MVP is done. Move to v1.1. No scope creep allowed.

## The API Key's 120-Second Life

- The user's Anthropic key lives in Redis for at most 120 seconds, encrypted with a per-job ephemeral key derived via HKDF.
- A Redis snapshot alone gives you ciphertext. You'd also need the master key, which lives in a separate env var with its own blast radius.
- The key never touches Postgres. Never in S3. Never in a structured log. Never in an exception traceback. Never in a URL or header.
- If a user reports a leak, the forensic answer is *"we don't have it; check Anthropic's audit log."*
- The doc names this cost in writing: *"Polite but unhelpful — accepted."* They signed off on the limitation rather than hiding it.

## The Override System That Defends Itself

- Custom params get rendered as a labelled bullet list in the system prompt. They are not concatenated into the user message.
- `tone=ignore_safety_guidelines` becomes a labelled scope hint, not a competing instruction. Structural defense beats brittle regex.
- The architecture itself is the safety substrate. They give it a property number, A1: *"the opinion IS the product."*
- They explicitly reject disabling Power Mode. Lock the namespace, keep the surface. New keys get telemetry-promoted, not user-invented.
- At MVP only ten vetted custom keys are allowed: `tone`, `pedagogy`, `citation_style`, `length_cap`, `audience_aside`, `examples_from`, `analogy_register`, `formality`, `bias_disclosure`, `verification_links`.

## What Got Cut From the Parent Product

- This started as a sibling to StudyBuddy_OnDemand. The interesting list is what they refused to carry across.
- Auth0. Three-track auth. School auth. RLS. Multi-tenant DB. Out.
- COPPA. FERPA. Stripe webhooks. Content review queue. AlexJS gating. Format-drift blocking. Out.
- "Adult-only product" turned out to be a subtraction tool, not just a positioning slide.
- The remaining footprint shrunk so much it became a separate repo, not a feature flag inside the parent.

## One-Sentence Takeaway

This is what a product looks like when a team treats refusal as a feature: every locked decision is also a defended boundary.

## If You Only Have 2 Minutes

- The whole pitch is refusing free-form chat. Six typed scope fields, never a chatbot mode.
- BYOK isn't just economics. It's a filter that excludes casual users by making them paste a real API key.
- Vendoring beat submodules and packages because release-cadence coupling would have killed two products that need to diverge.
- The user's Anthropic key lives in Redis for 120 seconds, encrypted, and never anywhere else. Period.
- Custom parameters are key:value blocks rendered as labelled scope hints, never concatenated into the user message.
- "Educational laundering" is a named threat. So is reputational blast-back from a banned BYOK key landing on the brand.
- The MVP doc explicitly lists what it's allowed to be fragile about. Most teams hide that list.
