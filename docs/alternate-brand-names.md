# Alternate Brand Names — StudyBuddy Q

> ⚠️ This is an **exploratory** document. The locked brand is **StudyBuddy Q**
> (see `SCOPE.md` §5, decision D19). Names below are alternatives generated
> for future reconsideration — none has been adopted, none has been
> trademark-screened. Do not link this from marketing, app-store listings, or
> external docs.

---

## 1. Naming brief — what the name has to carry

- **Adult self-learners**, not kids — soften or drop "Buddy" cuteness if we leave the family
- **Scoped-query is the IP** — six dimensions parameterise every lesson (topic, level, language, prior knowledge, format, real-world framing)
- **BYOK, premium feel** — Claude-Code-for-learners energy
- **Avoid Amazon Q collision** and bare single-letter trademarks
- **Distribution:** Google Play, App Store, .app domain — name must clear all three

---

## 2. Option A — Stay in the StudyBuddy family (sibling SKU)

Cheap on brand equity, expensive on "are you a kids' product?" perception.

| Candidate | Rationale | Caveat |
|---|---|---|
| **StudyBuddy Scope** | Names the IP directly; descriptive | Pedestrian; "Scope" alone is heavily used |
| **StudyBuddy Lens** | Scoping metaphor, softer than Scope | Apple Visual Lookup / Google Lens dilution |
| **StudyBuddy Atlas** | "World of knowledge" framing from CLAUDE.md | Atlassian collision in tech-adjacent space |
| **StudyBuddy Studio** | Signals craft + adult tooling (cf. Android Studio) | Dev-tool connotation may mislead audience |
| **StudyBuddy Solo** | Names the audience: solo learners, not classrooms | "Solo" feels limiting if we ever add multi-user |

---

## 3. Option B — Standalone, concept-led

Departs from the parent brand. Stronger TM defensibility, cleaner adult positioning, costs brand build.

| Candidate | Rationale | Caveat |
|---|---|---|
| **Facet** | Six scope dimensions = six facets of a query. On-the-nose for the IP, short, memorable | Common English word — TM scope will be narrow |
| **Aperture** | Opening through which knowledge passes; you adjust it (level, prior knowledge, format) before each shot. Photography metaphor reads premium | Aperture Science (Portal franchise) cultural collision |
| **Prism** | Splits one topic into the six dimensions like white light into colour. Visual, evocative | Many small SaaS already use Prism — TM crowding |
| **Codex** | Reference-book authority; pairs naturally with "Claude Code for learners" | OpenAI Codex (older product, mostly retired) — watch |
| **Sherpa** | Guide up an unfamiliar mountain. Warm, adult, not childish | Cultural-appropriation sensitivity in some markets |

---

## 4. Option C — Verb / posture names

| Candidate | Rationale | Caveat |
|---|---|---|
| **Grok** | Instant comprehension | Heinlein origin; xAI Grok dilution — likely unworkable |
| **Distill** | What the scoped query does to the LLM's output | Datadog Distillery, distill.pub — crowded |
| **Crystallize** | Same idea as Distill, more poetic | Less brandable; long; .com taken |

---

## 5. Ranked top 5 (with caveats)

1. **Facet** — best alignment with the IP; short; ".app" likely findable
2. **Aperture** — premium tone; Portal/Aperture Labs is the main collision risk
3. **StudyBuddy Studio** — safest if we want to keep parent equity
4. **Codex** — adult, authoritative; collides with OpenAI Codex (older product) — watch
5. **Prism** — evocative; many small SaaS products already use it — TM check needed

---

## 6. What still needs to happen before any of these moves

1. **USPTO TESS search** for each candidate (live mark, dead mark, related class 9 / 41 / 42)
2. **Google Play + App Store** name search
3. **Domain availability** — `.app`, `.com`, `.io`
4. **Amazon Q collision check** — explicit watch-out per `SCOPE.md` line 377
5. **Pronounceability test** in `en` / `fr` / `es` (D2: language scope)
6. **Logo-safe check** — does the name read well in the app icon at 48px?

Until these all clear, the locked brand remains **StudyBuddy Q** (D19).
